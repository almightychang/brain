# brain:thread — 주제 흐름 발견 및 관리

## 전제 조건

이 스킬 실행 전 `${CLAUDE_PLUGIN_ROOT}/config.md`를 읽어 VAULT, WORKSPACES 경로를 확인한다.

기존 thread와 최근 추가된 리소스(논문, 레포 등)를 스캔하여 연결 지점을 발견하고 추천한다.
인자 없이 호출. 연결이 없으면 과감하게 "없음"으로 끝낸다.

---

## Step 1: 기존 thread 스캔

```
VAULT="{config.md의 VAULT}"
```

`${VAULT}/6. Thread/` 에서 thread 문서를 찾는다:
- Glob: `**/*.md` (path: `${VAULT}/6. Thread`)
- 각 thread의 제목, 주제, 포함된 리소스 목록을 파악 (Read로 확인)

thread가 0개여도 괜찮다 — Step 3에서 새 thread 생성 추천이 가능하므로.

---

## Step 2: 최근 리소스 스캔

### 2-1. 논문

`${VAULT}/3. Resources/papers/` 에서 논문 summary(foldernote)를 스캔한다.

- Bash로 최근 수정된 논문 폴더 확인 (최근 2주 이내, 또는 최근 10개)
- 각 논문의 summary에서 핵심 주제/키워드/분야를 빠르게 파악
  - frontmatter의 tags, 한줄 요약, 핵심 기여 섹션을 읽는다
  - 전체를 읽을 필요 없음 — 상단 50줄이면 충분

### 2-2. 레포

`${VAULT}/3. Resources/repos/` 에서 레포 foldernote를 스캔한다.

- Bash로 최근 수정된 레포 폴더 확인 (최근 2주 이내, 또는 최근 10개)
- 각 레포의 foldernote에서 핵심 주제/기능을 빠르게 파악
  - frontmatter의 tags, 한줄 요약, 왜 아카이빙했는가 섹션을 읽는다
  - 상단 50줄이면 충분

---

## Step 3: 연결 지점 분석 및 추천

### 3가지 케이스

**Case A: 기존 thread에 추가할 리소스가 있음**
- 리소스의 주제가 기존 thread의 문제 정의/방향과 겹침
- 추천: "{{리소스}}를 {{thread}}에 추가할 수 있을 것 같습니다. {{이유}}."

**Case B: 새 thread를 만들 만한 리소스 클러스터가 있음**
- thread에 속하지 않는 리소스 2개 이상이 공통 주제로 묶임
- 리소스는 논문+논문, 논문+레포, 레포+레포 등 어떤 조합이든 가능
- 추천: "{{리소스1}}, {{리소스2}}가 '{{주제}}'로 묶일 수 있을 것 같습니다. thread를 만들까요?"

**Case C: 연결 지점 없음**
- 출력: "현재 연결할 만한 것이 없습니다."
- 끝. 억지로 만들지 않는다.

---

## Step 4: 사용자 승인 후 실행

사용자가 추천을 수락하면:

### 기존 thread에 리소스 추가 시

1. thread 문서를 Read
2. 추가할 리소스의 summary/foldernote를 Read
3. 리소스가 기존 분류(방향/서브섹션)에 해당하면 → 해당 테이블에 행 추가
4. 새로운 방향이면 → 새 서브섹션(###) 생성
5. "아직 없는 것", "핵심 긴장" 섹션도 필요시 업데이트
6. Edit 도구로 수정

### 새 thread 생성 시

thread 문서 위치: `${VAULT}/6. Thread/{{Thread 제목}}.md`

```markdown
---
created-by: claude-code
created-at: {{오늘 날짜}}
tags:
  - thread
  - {{주제 태그들}}
---

# {{Thread 제목}}

> 출발점: [[첫 번째 리소스의 summary/foldernote|리소스 제목]]

## 문제 정의

{{이 thread가 다루는 핵심 문제/질문을 2-3문단으로}}

## 현재 흐름 ({{연도 범위}})

### {{방향 1}}

{{설명}}

| 리소스 | 유형 | 날짜 | 핵심 아이디어 |
|--------|------|------|-------------|
| **{{이름}}** ([링크](URL)) | 📄/🔧 | {{날짜}} | {{한줄 요약}} |

### {{방향 2}}

...

## 아직 없는 것

{{현재 이 주제에서 빠져있는 부분}}

## 핵심 긴장

{{이 분야의 근본적인 trade-off}}

## 개인 관찰

{{사용자에게 물어서 채운다 — "이 주제에 대해 개인적인 관점이나 관심사가 있으신가요?"}}
```

**리소스 유형 아이콘:**
- 📄 논문 (paper)
- 🔧 레포 (repo)

---

## Step 5: 완료

```
✅ Thread {{생성됨/업데이트됨}}: {{파일 경로}}

📦 포함된 리소스: N개
  - 📄 [[논문1]]
  - 🔧 [[레포1]]
  - ...
```
