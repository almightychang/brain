# brain:project — 프로젝트 조회 & 탐색

## 전제 조건

이 스킬 실행 전 `${CLAUDE_PLUGIN_ROOT}/config.md`를 읽어 VAULT, WORKSPACES 경로를 확인한다.

## 서브커맨드

| 커맨드 | 설명 | 예시 |
|--------|------|------|
| `list` | 프로젝트 목록 한눈에 보기 | `/brain:project list` |
| `open [name]` | 프로젝트 폴더 탐색 + 핵심 요약 | `/brain:project open Data Platform` |

첫 번째 인자(`$ARGUMENTS[0]`)로 서브커맨드를 판별한다. 인자가 없으면 `list`를 실행한다.

# list — 프로젝트 목록 조회

Obsidian Second Brain의 `1. Project/` 폴더에 있는 프로젝트들을 한눈에 보여준다.

## 실행 절차

### 1. 프로젝트 폴더 스캔

`${VAULT}/1. Project/` 아래의 **1단계 하위 폴더** 목록을 가져온다.

### 2. 각 프로젝트 메인 노트 읽기

각 프로젝트 폴더에서 foldernote 규칙에 따라 **폴더명과 동일한 이름의 .md 파일**의 frontmatter와 첫 번째 섹션(개요/요약)만 읽는다. 본문 전체를 읽지 않는다.

### 3. 출력

다음 형식으로 출력한다:

```
## 프로젝트 목록

| # | 프로젝트 | 상태 | 한줄 요약 |
|---|---------|------|----------|
| 1 | 프로젝트명 | 진행중 / 대기 / 중단 / 완료 | 메인 노트에서 파악한 한줄 요약 |
| 2 | ... | ... | ... |

**총 N개 프로젝트** | 활성: X개, 대기: Y개, 완료: Z개
```

### 상태 판단 기준

메인 노트의 내용에서 상태를 추론한다:
- **진행중**: 진행 상황이 기술되어 있고 활발한 작업이 보이는 경우
- **대기**: "대기", "보류", "pending" 등의 표현이 있는 경우
- **중단**: "중단", "취소" 등의 표현이 있는 경우
- **완료**: "완료", "done", "finished" 등의 표현이 있는 경우
- 판단이 어려우면 `불명`으로 표시

# open — 프로젝트 리마인더

Obsidian Second Brain의 프로젝트 문서를 빠르게 훑어보고 핵심 컨텍스트를 리마인드한다.

## 입력

`/brain:project open <프로젝트 경로 또는 이름>`

### 경로가 주어진 경우
- 절대 경로 또는 `@` 참조 경로를 그대로 사용
- `1. Project/프로젝트명` 형태의 상대 경로면 `${VAULT}/` 아래로 해석

### 이름만 주어진 경우
- `${VAULT}/1. Project/` 아래에서 이름이 포함된 폴더를 검색
- 여러 개 매치되면 목록을 보여주고 선택 요청

### 아무것도 없는 경우
- `${VAULT}/1. Project/` 의 프로젝트 목록을 보여주고 선택 요청

## 실행 절차

### 1. 폴더 구조 파악

프로젝트 폴더의 전체 파일/디렉토리 구조를 트리 형태로 파악한다.

### 2. 메인 노트 읽기

foldernote 규칙에 따라 **폴더명과 동일한 이름의 .md 파일**이 메인 노트다. 이 파일을 읽는다.

### 3. 서브 노트 스캔

서브 폴더와 나머지 .md 파일들의 **제목(첫 번째 `#` 헤딩)과 frontmatter**만 빠르게 읽는다. 본문 전체를 읽지 않는다.

### 4. 출력

다음 형식으로 간결하게 출력한다:

```
## {프로젝트명}

**생성일**: YYYY-MM-DD | **태그**: tag1, tag2, ...

### 핵심 요약
{메인 노트의 핵심 내용을 3-5줄로 요약}

### 문서 구조
{트리 형태의 파일 구조 — 각 파일에 한줄 설명}

### 현재 상태
{메인 노트에서 파악 가능한 진행 상태, 열린 질문, 다음 단계 등}
```

---

## 원칙 (공통)

- **속도 우선**: 메인 노트 + 서브 노트 헤딩/frontmatter만 스캔. 전체 파일을 다 읽지 않는다.
- **간결하게**: 요약은 핵심만. 상세 내용이 궁금하면 사용자가 추가 질문.
- **위키링크 보존**: `[[]]` 링크는 그대로 표시하여 Obsidian 컨텍스트 유지.
- **판단 최소화**: 명시적 단서가 없으면 `불명`으로 표시. 추측하지 않는다.
- **정렬** (list): 상태 기준으로 정렬 (진행중 → 대기 → 불명 → 중단 → 완료)
